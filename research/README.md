# Research

## ZERO TRUST AECHITECTURE FOR CLOUD COMPUTING

Objective:

The main objective of this project is to examine the security challenges present in modern cloud computing environments and evaluate how Zero Trust Architecture (ZTA) can address these challenges. The project aims to analyze cloud security threats, including data breaches, insider attacks, API vulnerabilities, and AI-driven risks, while exploring the principles of Zero Trust such as continuous verification, least privilege access, and continuous monitoring. Through the review of selected research papers, the project seeks to demonstrate why traditional perimeter-based security models are no longer sufficient and how Zero Trust provides a more effective security framework for protecting cloud systems.

Results/Findings

The findings of this project show that cloud environments are increasingly vulnerable to both traditional and emerging cyber threats due to their distributed and interconnected nature. The reviewed papers revealed that security risks such as unauthorized access, data leakage, prompt injection attacks, and insecure APIs continue to challenge cloud security. The study found that Zero Trust Architecture significantly improves security by eliminating implicit trust and requiring continuous authentication and authorization for every user, device, and access request. The results also indicate that while Zero Trust enhances protection against modern threats, challenges such as implementation complexity, scalability issues, and lack of standardization still need to be addressed. Overall, the project concludes that Zero Trust Architecture is an effective and essential approach for securing modern cloud computing environments.
